rest

The rest provider is used to fetch a resource from an API endpoint.

Example configuration​

fetcher:
  providers:
    api:
      rest:
        headers:
          Content-Type: application/json
          Authorization: "Bearer: my_jwt_token"

loader:
  rules:
    load-resource:
      raw:
        resource:
          method: GET
          url: https://<backend_url>/api/v1/resource
          params:
            limit: 10
  • The provider api uses the provider module rest.
  • The loader rules has a rule for the resource resource which loads it using the provider api with the given configuration.

Directives​

tlsCAFiles​

    Syntax:         tlsCAFiles:
    Type:           list
    List item:      string
    Default:        -

The TLS CA file(s) of the fetch client.

tlsCertFiles​

    Syntax:         tlsCertFiles:
    Type:           list
    List item:      string
    Default:        -

The TLS client certificate file(s) of the fetch client.

tlsKeyFiles​

    Syntax:         tlsKeyFiles:
    Type:           list
    List item:      string
    Default:        -

The TLS client key file(s) of the fetch client.

timeout​

  Syntax:         timeout: <duration>
  Type:           numeric
  Unit:           second
  Default:        15

The timeout to fetch a request.

connectTimeout​

  Syntax:         connectTimeout: <duration>
  Type:           numeric
  Unit:           second
  Default:        5

The connect timeout to fetch a request.

maxConnsPerHost​

    Syntax:         maxConnsPerHost: <count>
    Type:           numeric
    Unit:           connections
    Default:        100

The maximum number of connections per host.

A value of 0 means no limit.

maxIdleConns​

    Syntax:         maxIdleConns: <count>
    Type:           numeric
    Unit:           connections
    Default:        100

The maximum number of idle connections.

A value of 0 means no limit.

maxIdleConnsPerHost​

    Syntax:         maxIdleConnsPerHost: <count>
    Type:           numeric
    Unit:           connections
    Default:        100

The maximum number of idle connections per host.

A value of 0 means no limit.

idleConnTimeout​

    Syntax:         idleConnTimeout: <duration>
    Type:           numeric
    Unit:           second
    Default:        60

The timeout of idle connections.

A value of 0 means no limit.

retry​

    Syntax:         retry: <count>
    Type:           numeric
    Default:        3

The maximum number of retries for all requests.

A value of 0 means no retry.

retryDelay​

    Syntax:         retryDelay: <duration>
    Type:           numeric
    Unit:           second
    Default:        1

The delay between a retry.

A value of 0 means no delay.

headers​

    Syntax:         headers:
    Type:           map
    Map key:        string
    Map value:      string

The global headers to use for requests.

Example:

headers:
  header1: value1
  header2: value2

params​

    Syntax:         params:
    Type:           map
    Map key:        string
    Map value:      string
    Default:        -

The global parameters to use for requests.

Example:

params:
  param1: value1
  param2: value2

Resource configuration​

method​

    Syntax:         method: <verb>
    Type:           string
    Valid values:   GET, POST, PATCH, PUT, DELETE, HEAD, OPTIONS
    Default:        -

The request method.

url​

    Syntax:         url: <url>
    Type:           string
    Default:        -

The request URL.

headers​

    Syntax:         headers:
    Type:           map
    Map key:        string
    Map value:      string
    Default:        -

The request headers.

tip

Any existing global header will be overwritten.

Example:

headers:
  header1: value1
  header2: value2

params​

    Syntax:         params:
    Type:           map
    Map key:        string
    Map value:      string
    Default:        -

The request parameters.

tip

Any existing global parameter will be overwritten.

Example:

params:
  param1: value1
  param2: value2
